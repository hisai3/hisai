Webstrom 常用快捷配置

插件

String Manipulation 字符串处理 驼峰、下划线。。。

常用快捷

（修改html注释样式 code style –> html –> code generation 想的勾选都去除）
ctrl + / : 单行注释

ctrl + shift + / : 多行注释

ctrl + alt + s 打开设置

ctrl + d :

  1. 光标处于文件内容中，复制选择内容 或者 复制当前行

  2. 鼠标选中文件，选择对比文件

shift + f6 重命名

alt + f7 ： 选中文件，查找有引入选中文件的文件

alt + click ： 选中多个位置同时编辑

ctrl + shift + v ： 剪贴板

f4 : 在git下选择文件后定位到文件位置

自定义快捷方式

File Path 文件本地目录结构（edit–复制文件绝对路径ctrl+shift+c nav–查看项目本地文件夹 f7）

alt + shift + f : Reformat Code 格式化
(将js 双引转单引 ‘Code Style - javascript - Punctuation’ 全部设置always， 第一个设置don’t use)

ctrl + h : Show History 对比当前文件的上一次历史修改（右键 – local history）

Show History From Selection 对比当前选择中的上一次历史修改（右键 – local history） ctrl + h

ctrl + 1 : 文件结构 File Structure

crtl + shift + b : 选中分支比较文件 compare with branch

crtl + shift + k : git push

alt + shift + v : 选中commit比较文件 compare with version

alt + shift + p : git pull

alt + shift + m : git merge

alt + shift + c : git stash

alt + shift + u : git stash apply

alt + shift + r : git reset

ctrl + w : close 关闭当前tag

alt + w : close 关闭所有tag

alt + o : close 关闭其他tag

Vue项目@引入跳转问题

setting 搜索webpack 选择language..下的添加webpack.config.js(可以直接选择项目的)

最好将 webpack 按项目的走（设置automatically）

关于webstorm打开Vue项目时，无法通过Ctrl+鼠标左键快速定位的解决方法，解决打开.vue文件时打开一直卡在Analyzing的状态

原因: node_module太多文件
解决：file types忽略node_module

SVN配置

  1. 安装svn时
  2. webstorm配置

添加svn远程仓库 设置中填写仓库地址即可

svn与git一些区别

操作 svn git
忽略提交 右键文件(夹) -> tortoiseSvn -> showlog -> add to ignore list 新分支
初次下载项目 tortoiseSvn -> export -> 填入仓库地址 -> 点击…预览文件 -> 点击选择项目 -> ok (注意：这样下载只会下载选中文件夹下的文件, 所以要先在本地新建一个跟项目名一样的文件夹) clone
提交代码 commit push
分支 相当于复制文件夹 tortoiseSvn -> branch/tag check -b
回退 tortoiseSvn -> showlog -> 右键revert… checkout/reset/revert
暂存 create patch(将修改的文件生成.pacth文件, 使用apply patch恢复, 注意：生成之后原代码中还是会有修改的代码) stash (恢复: apply 恢复并删除：pop)
